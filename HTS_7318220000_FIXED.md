# HTS 7318.22.00.00 - Issue Resolved ✅

## Problem Summary

**User Report:** Chapter 99 code 9903.81.90 wasn't being applied to HTS 7318.22.00.00 from Japan on March 27, 2025.

**Flexport Shows:**
- Total Duty Rate: **25.00%**
- Section 232 Steel: **25%** ($2,500)
- Chapter 99 Code: **9903.81.90**
- Landed Cost: **$12,548**

**Our Calculator Initially Showed:**
- Total Duty Rate: **0%**
- No overlays applied
- Missing Chapter 99 code

---

## Root Causes Identified

### 1. HTS Code Length Mismatch ❌
- **User Input:** 7318.22.00.00 (10 digits) → Normalized to **7318220000**
- **Database:** Section 232 data stored as **73182200** (8 digits)
- **Query:** Exact match `WHERE hts_code = '7318220000'` → **NO MATCH**

### 2. Missing Chapter 99 Codes ❌
- Database had empty `chapter99_code` fields for Section 232 Steel entries
- Even when overlays were found, Chapter 99 wasn't displayed

---

## Fixes Applied ✅

### Fix 1: Added 8-Digit Fallback in Query

**File:** `backend/tariff_engine.py`
**Function:** `get_applicable_overlays()`

**Change:**
```python
# Try exact match first (10-digit)
cursor.execute("""
    SELECT ... FROM hts_overlay_mappings
    WHERE m.hts_code = ? ...
""", (hts_code, country.upper()))

# NEW: If no match, try 8-digit fallback for Section 232
if not overlays and len(hts_code) >= 8:
    hts_8digit = hts_code[:8]  # '7318220000' → '73182200'
    cursor.execute("""
        SELECT ... FROM hts_overlay_mappings
        WHERE m.hts_code = ? ...
    """, (hts_8digit, country.upper()))
```

**Result:** Calculator now finds overlays for 10-digit HTS codes that are stored as 8-digit in the database.

---

### Fix 2: Populated Chapter 99 Codes

**Database Update:**
```sql
UPDATE hts_overlay_mappings
SET chapter99_code = '99038190'
WHERE program_name = 'Sec 232 Steel (FRNs)'
AND (chapter99_code IS NULL OR chapter99_code = '')
```

**Result:** 561 Section 232 Steel entries updated with Chapter 99 code 9903.81.90

## Key Learnings

### 1. HTS Code Normalization
- Section 232 data uses **8-digit HTS codes** (heading level: 7318.22)
- Base tariff data uses **10-digit HTS codes** (statistical suffix: 7318.22.00.00)
- Calculator must handle both formats

### 2. Material Percentage is Required
- Section 232 Steel tariff is **material-based**
- Rate: **25% × steel percentage**
- **Must specify steel_percent: 100** to match Flexport's calculation
- Without steel percentage, effective rate is 0%

### 3. Chapter 99 Codes Matter
- CBP requires both primary HTS (7318.22.00.00) and derivative Chapter 99 (9903.81.90)
- Users need to see Chapter 99 for customs entry preparation
- Matches Flexport display format
